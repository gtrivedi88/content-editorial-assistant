= Environment variables reference
:description: Complete reference for all environment variables in Content Editorial Assistant
:keywords: configuration, environment, variables, setup

[NOTE]
====
**This documentation is for developers and system administrators** who are deploying or configuring Content Editorial Assistant locally or in production environments.
====

== Overview

Content Editorial Assistant uses environment variables for all configuration. Variables are read in `app/config.py` via `os.environ.get()` with sensible defaults. A `.env` file in the project root is loaded automatically via `python-dotenv`.

== Configuration methods

=== 1. `.env` file (recommended for development)

Create a `.env` file in the project root:

[source,bash]
----
MODEL_PROVIDER=ollama
BASE_URL=http://localhost:11434
MODEL_ID=llama3:8b
MODEL_ANALYSIS_TEMPERATURE=0.0
MODEL_SEED=42
----

=== 2. Environment variables (production)

Export variables in your shell or container environment:

[source,bash]
----
export MODEL_PROVIDER=api
export BASE_URL=https://api.groq.com/openai/v1
export ACCESS_TOKEN=your-token-here
----

=== 3. GitLab CI/CD variables

For GitLab CI/CD, add variables at **Settings → CI/CD → Variables**:

* Mark sensitive keys as **Masked**
* Mark production keys as **Protected**

== LLM / Model provider

[cols="2,1,3"]
|===
|Variable |Default |Description

|`LLM_ENABLED`
|`True`
|Enable or disable LLM-based analysis

|`MODEL_PROVIDER`
|`llamastack`
|AI provider: `ollama`, `api`, or `llamastack`

|`MODEL_ID`
|_(provider-specific)_
|Model identifier for the selected provider

|`MODEL_TEMPERATURE`
|`0.4`
|Default sampling temperature (fallback)

|`MODEL_ANALYSIS_TEMPERATURE`
|`0.0`
|Temperature for analysis (greedy decoding for consistency)

|`MODEL_SUGGESTION_TEMPERATURE`
|`0.2`
|Temperature for suggestion generation (slightly creative)

|`MODEL_SEED`
|`42`
|Seed for reproducible generation (best-effort per provider)

|`MODEL_MAX_TOKENS`
|`3072`
|Maximum output tokens per generation

|`LLM_MAX_CONCURRENT`
|`5`
|Max concurrent LLM requests (ThreadPoolExecutor workers)

|`LLM_GLOBAL_PASS_MAX_WORDS`
|`5000`
|Word-count ceiling for the global LLM pass

|`LLM_EXCERPT_BUDGET_MAX`
|`8000`
|Token budget for style guide excerpts in LLM prompts

|`BLOCK_CACHE_TTL`
|`3600`
|Block-level cache time-to-live in seconds

|`LLM_JUDGE_ENABLED`
|`False`
|Enable optional second-pass LLM quality filter
|===

== Example configurations

LOG_FORMAT=json

== Security best practices

1. **Never commit `.env` files** — add to `.gitignore`
2. **Use masked variables in CI/CD** — prevent exposure in logs
3. **Rotate API keys regularly** — update `ACCESS_TOKEN` periodically
4. **Set a strong `SECRET_KEY`** — generate with: `python -c "import secrets; print(secrets.token_hex(32))"`
5. **Restrict CORS origins** — do not use `*` in production
